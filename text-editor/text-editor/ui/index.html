Text Editor › Core › UI

This is a module to create the GUI appearance as you can see in the text editor below.

Destroy · Create · Disable · Enable · Freeze · Trigger Bold Button · Disable Bold Button · Enable Bold Button · Move Bold Button to 2 from Left · Move Bold Button to 2 from Right · Move Bold Button to 1 from Left · Move Bold Button to 1 from Right

Start

Include ui.min.css and ui.i.min.css in the head section, include text-editor.min.js and ui.min.js next to the text area:

<!DOCTYPE html>
<html dir="ltr">
  <head>
    <meta charset="utf-8">
    <title>Test</title>
    <link href="text-editor/ui/ui.min.css" rel="stylesheet">
    <link href="text-editor/ui/ui.i.min.css" rel="stylesheet">
  </head>
  <body>
    <p><textarea></textarea></p>
    <script src="text-editor.min.js"></script>
    <script src="text-editor/ui/ui.min.js"></script>
    <script>
    var editor = TE.ui(document.querySelector('textarea'));
    </script>
  </body>
</html>

Configure

Instance

TE.ui(node, config);
  • node → The text area element.
  • config → Custom configuration data to override the default configuration data.
var editor = TE.ui(document.querySelector('textarea'), { … });

Configuration

The available configuration data are:

  • tab → string · Set custom indent character.
  • direction → string · Editor language direction (currently this option only set a HTML dir attribute to the text editor container).
  • path → string · Directory path of text-editor.min.js file.
  • resize → boolean · Enable/disable text area resize handler.
  • keys → boolean · Enable/disable keyboard shortcut feature.
  • tools → string · List of tools to be included to the tools bar, each of them separated by a space.
  • attributes → object · Extra HTML attributes to be added to the text area element.
  • auto_tab → boolean · Press ⇥ key to indent?
  • auto_close → object · List of character as the object key that will be auto–closed by its value right after the text area caret.
  • languages → object · List of words/phrases used by the text editor GUI.
  • classes → object · List of HTML classes used by the text editor GUI.

Custom themes can be included just after the ui.min.css:

<link href="text-editor/ui/ui.min.css" rel="stylesheet">
<link href="text-editor/ui/ui/ui.white.min.css" rel="stylesheet">

Keyboard Shortcuts

  • ⌘+y → Redo
  • ⌘+z → Undo
  • F10 → Focus Tools. Focus to the first tool item in the tools bar. Use arrow keys to navigate between tools.
  • ⎋ → Release Focus. Release editor focus by focusing on the next focusable element.
  • ⇥ → Indent
  • ⇧+⇥ → Outdent

Tidy White–Spaces

Smarter method to force white–spaces between selection. This method will not add white–spaces at the start of text selection if characters before selection is empty, and will not add white–spaces at the end of text selection if characters after selection is empty:

editor.tidy(gap_before, gap_after);
  • gap_before → Gap before selection.
  • gap_after → Gap after selection.
editor.tidy('\n\n', '\n\n'); // force block

Toggle Wrap

editor.format(tag, gap_outer, gap_inner);
  • tag → The tag name. You can also add attributes there.
  • gap_outer → Gap before and after tag boundaries.
  • gap_inner → Gap before and after selection.
editor.format('strong');
editor.format('time datetime="2016-21-04"');
editor.format('h1', '\n\n', '\n'); // force block

Components

Set callback function on the optional fn parameter.

Icon

Create an icon:

editor.ui.i(id, alias);
  • id → Icon ID.
  • alias → Semantic icon name.
editor.ui.i('pencil', 'edit'); // return `<i class="text-editor-i text-editor-i-pencil text-editor-i-edit"></i>

Tool test

editor.ui.tool(id, data, index);
  • id → Tool ID.
  • data → Tool data:
    • title → Tool title.
    • i → Icon identifier or key alias.
    • click (e, editor, id) → Function that will be executed on click event.
    • text → Use text instead of icon?
    • active → true by default, it means that the tool is active and can be used.
    • on → Add an on class to the tool markup.
    • attributes → Extra HTML attributes for the tool markup.
    • target → HTML element used to trigger the tool item (generated automatically).
  • index → Tool index placement in the tools bar (optional).
editor.ui.tool('emoticon', {
    i: 'smile-o',
    title: 'Emoticon',
    click: function(e, $, id) {
        // `e` refers to the `HTMLEvents` instance
        // `$` refers to the `TE` instance
        // `id` refers to `emoticon` where possible
        return $.insertBefore(' :) '), false;
    }
}, index);

A tool without i property will use the tool key as the icon ID:

// this…
editor.ui.tool('bold', {
    click: function(e, $) { … }
});

// is equal to this…
editor.ui.tool('bold', {
    i: 'bold',
    click: function(e, $) { … }
});

A tool with function value will use that function as the click property:

// this…
editor.ui.tool('bold', function(e, $) { … });

// is equal to this…
editor.ui.tool('bold', {
    i: 'bold',
    click: function(e, $) { … }
});

Return the added tools:

console.log(editor.ui.tools);

Trigger a tool item. This is the same as clicking the indent tool:

editor.ui.tools.indent.click(null, editor);

Trigger a menu item. This is the same as clicking the justify menu item from align tool:

editor.ui.tools.align.data.justify.click(null, editor);

Overlay test

editor.ui.overlay(id, click_to_exit, fn, hook);
  • id → Overlay ID.
  • click_to_exit → true to remove the overlay by clicking on it.
  • fn → Callback function.
  • hook → Hook namespace.
editor.ui.overlay(true, fn);

Panel test

editor.ui.panel(id, node, fn, hook);
  • id → Panel ID.
  • node → Return value of TE._.el or a HTML string or a function or an object contains header (as panel title) and body (as panel content) property.
  • fn → Callback function.
  • hook → Hook namespace.
editor.ui.panel('my-panel', 'Panel content.', fn);
editor.ui.panel('my-panel', function(panel) {
    panel.innerHTML = 'Panel content.';
}, fn);
editor.ui.panel('my-panel', {
    header: 'Panel Title',
    body: 'Panel content.'
}, fn);

Modal test

editor.ui.modal(id, pane, fn, hook);
  • id → Modal ID.
  • pane → Modal content:
    • header → Modal header.
    • body → Modal body.
    • footer → Modal footer.
  • fn → Callback function.
  • hook → Hook namespace.
editor.ui.modal('my-modal', {
    header: 'Modal Header',
    body: 'Modal body.',
    footer: 'Modal footer.'
}, fn);
var button_1 = TE._.el('button', 'OK'),
    button_2 = TE._.el('button', 'Cancel');

editor.ui.modal('my-modal', {
    header: 'Modal Header',
    body: 'Modal body.',
    footer: [button_1, button_2]
}, fn);
editor.ui.modal('my-modal', {
    header: 'Modal Header',
    body: 'Modal body.',
    footer: {
        'OK': function(e, $) {
            alert('ok');
        },
        'Cancel': function(e, $) {
            alert('cancel');
        }
    }
}, fn);

The example below will replace okay and cancel key with editor.config.languages.buttons.okay and editor.config.languages.buttons.cancel value if available, otherwise, it will return the key anyway as the button text:

editor.ui.modal('my-modal', {
    header: 'Modal Header',
    body: 'Modal body.',
    footer: {
        okay: function(e, $) { … },
        cancel: function(e, $) { … }
    }
}, fn);

The example below will replace okay and cancel key with its text property as the button text:

editor.ui.modal('my-modal', {
    header: 'Modal Header',
    body: 'Modal body.',
    footer: {
        okay: {
            text: 'OK',
            click: function(e, $) { … }
        },
        cancel: {
            text: 'Cancel',
            click: function(e, $) { … }
        }
    }
}, fn);

Exit

Hide panel, overlay, modal, drop and bubble.

editor.ui.exit(restore_selection, key);
  • restore_selection → Restore selection after panel, overlay, modal, drop and bubble disappear.
  • key → Exit specific component.
editor.ui.exit(true);
editor.ui.exit(true, 'modal'); // exit modal only
editor.ui.exit.panel(restore_selection); // hide panel only
editor.ui.exit.overlay(restore_selection); // hide overlay only
editor.ui.exit.modal(restore_selection); // hide modal only
editor.ui.exit.drop(restore_selection); // hide drop only
editor.ui.exit.bubble(restore_selection); // hide bubble only

Fit

Re–position the components.

editor.ui.modal.fit(); // re–center the modal
editor.ui.modal.fit([2, 4]); // set `x` offset to `2` and `y` offset to `4`
editor.ui.drop.fit(); // re–snap the drop
editor.ui.drop.fit([2, 4]); // set `x` offset to `2` and `y` offset to `4`
editor.ui.bubble.fit(); // re–snap the bubble
editor.ui.bubble.fit([2, 4]); // set `x` offset to `2` and `y` offset to `4`

Key

Keyboard shortcuts.

editor.ui.key(keys, fn);
  • keys → A lower–cased KeyboardEvent.key value or multiple KeyboardEvent.key value separated by + sign.
  • fn (e, editor) → Function to execute on keyboard keys combination match. Set as false to remove the keys combination from editor.ui.keys.
editor.ui.key('control+b', function(e, $) {
    return $.wrap('<b>', '</b>'), false;
});

A key with string value will be used to take a tool item’s click property from the editor.ui.tools if available:

// this…
editor.ui.key('control+b', 'b');

// is equal to this…
editor.ui.key('control+b', editor.ui.tools.b.click);

Overlay › Message test

editor.ui.message(id, node, auto_exit, fn, hook);
  • id → Message ID.

  • auto_exit → Delay in milliseconds to automatically remove the message bar. Set to 0 or false to disable.

editor.ui.message('my-message', 'Message content.', 3000);

Modal › Alert

editor.ui.alert(header, body, fn);
  • header → Alert title.
  • body → Alert description.
  • fn (e, editor) → Function to execute when clicking on the Okay button.
editor.ui.alert('Alert Title', 'Alert body.', function(e, $) {
    console.log('ok');
});

Modal › Confirm

editor.ui.confirm(header, body, fn_okay, fn_cancel);
  • header → Confirm title.
  • body → Confirm description.
  • fn_okay (e, editor) → Function to execute when clicking on the Okay button.
  • fn_cancel (e, editor) → Function to execute when clicking on the Cancel button.
editor.ui.confirm('Confirm Title', 'Confirm body.', function(e, $) {
    console.log('ok');
}, function(e, $) {
    console.log('cancel');
});

Modal › Prompt

editor.ui.prompt(header, placeholder, required, fn, type);
  • header → Prompt title.
  • placeholder → Prompt input placeholder (or default value if required is false).
  • required → Set to true to force user to change the input value other than value and empty value to make it possible to submit the prompt.
  • fn (e, editor, value, placeholder) → Function to execute when input value is valid.
  • type → Prompt input type (optional, default is 0):
    • 0 → <input name="data" type="text"> test
    • 1 → <textarea name="data"></textarea> test
    • 2 → <select name="data"> … </select> test
editor.ui.prompt('Prompt Title', 'Prompt value…', true, function(e, $, value, placeholder) {
    console.log(arguments);
}, type);
function is_valid_url(value) {
    return /^https?:\/\/\S+$/.test(value);
}
editor.ui.prompt('Prompt Title', 'Prompt value…', is_valid_url, function(e, $, value, placeholder) {
    console.log(arguments);
}, type);

If type parameter is set to 2, use placeholder as the select box options container, then set the selected value in required parameter:

editor.ui.prompt('Choose', {
    'red': 'Red',
    'green': 'Green',
    'blue': 'Blue'
}, 'green', function(e, $, value, placeholder) {
    $.insert(value);
}, 2);

Drop › Menu

editor.ui.menu(id, selected, data, index);
  • id → Tool ID.
  • selected → A key from data to select, or use any string as placeholder.
  • data → Object of keys and values, where value is an object of editor.ui.tool’s data or a string or a function.
  • index → Tool index placement in the tools bar (optional)
editor.ui.menu('my-menu', 'p', {
    h1: function(e, $) { … },
    h2: function(e, $) { … },
    h3: function(e, $) { … },
    h4: function(e, $) { … },
    h5: function(e, $) { … },
    h6: function(e, $) { … },
     p: function(e, $) { … }
}, index);
editor.ui.menu('my-menu', 'b', {
    b: 'b',
    i: 'i',
    u: 'u',
    s: 's'
}, index);

A key with string value will be used to take a tool item from the editor.ui.tools if available:

// this…
editor.ui.menu('my-menu', 'b', {
    i: 'i'
});

// is equal to this…
editor.ui.menu('my-menu', 'b', {
    i: editor.ui.tools.i
});

A click property with string value will be used to take a tool item’s click property from the editor.ui.tools if available:

// this…
editor.ui.menu('my-menu', 'b', {
    i: {
        click: 'i'
    }
});

// is equal to this…
editor.ui.menu('my-menu', 'b', {
    i: {
        click: editor.ui.tools.i.click
    }
});

selected value as array will take the first array item as icon identifier and the second array item as key of data item to select or as the string itself:

editor.ui.menu('my-menu', ['bold', 'b'], {
    b: {
        text: ['bold', 'Bold'],
        click: function(e, $) { … }
    }
}, index);

Programmatically set the selected menu item:

editor.ui.menu.set(id, selected);
editor.ui.menu.set('my-menu', 'h2'); // select `h2`

Utilities

Added more helper functions to the TE._ object:

Time

Get time data as year, month, date, hour, minute, second and millisecond:

console.log(TE._.time()); // return `["2016", "11", "26", "17", "25", "23", "258"]`

Replace

Multiple replace function using array:

var from = ['&', '<', '>'];
var to = ['&amp;', '&lt;', '&gt;'];
var encoded = TE._.replace(input, from, to);

The string array data is always global. Therefore, you can also use regular expression as the from array data:

TE._.replace(input, [/\s+/g, '&'], ["", '&amp;']);

Format

Replace pattern in a string with custom text. A pattern is numbers prefixed by a %. Each number will be replaced by data item from an array based on its matching index (1–based index):

var s = TE._.format('Lorem %1 ipsum dolor %2 sit amet.', ['apple', 'orange']);

DOM

Create a new element (the node):

var node = TE._.el('div'); // return `<div></div>`
var node = TE._.el('div', 'test'); // return `<div>test</div>`
var node = TE._.el('div', 'test', {
    'id': 'foo',
    'class': 'bar'
}); // return `<div id="foo" class="bar">test</div>`

Special attributes are classes, data and css:

TE._.el('div', "", {
    'classes': ['foo', 'bar', 'baz']
}); // return `<div class="foo bar baz"></div>`
TE._.el('div', "", {
    'data': {
        'foo': 0,
        'bar': 1,
        'baz': 2
    }
}); // return `<div data-foo="0" data-bar="1" data-baz="2"></div>`
TE._.el('div', "", {
    'css': {
        'background': '#fff',
        'color': '#000'
    }
}); // return `<div style="background: #fff; color: #000;"></div>`

DOM Helpers

Helper functions for DOM manipulation. Asumming node and container is a return value of TE._.el.

Set

Append node to container:

TE._.dom.set(container, node);

Reset

Remove node from container:

TE._.dom.reset(container, node);

Get

Get node from container context:

TE._.dom.get(node, container); // check if `node` is a child of `container`
TE._.dom.get('#foo', container); // is equal to `[container.getElementById('foo')]`
TE._.dom.get('.foo', container); // is equal to `container.getElementsByClassName('foo')`
TE._.dom.get('#foo .bar', container); // is equal to `container.querySelectorAll('#foo .bar')`

If context is not defined, it will be referenced to the document.

Is

Check if node is an <i> tag:

if (TE._.dom.is(node, 'i')) { … }

ID

Setting a unique ID to node if it does not have an id attribute:

TE._.dom.id(node);

Copy

Clone node:

var clone = TE._.dom.copy(node);

Replace

Replace node with foo:

var foo = TE._.el('strong');
TE._.dom.replace(node, foo);

Class

Add/remove/toggle and check HTML classes:

TE._.dom.classes.set(node, 'active');
TE._.dom.classes.reset(node, 'active');
TE._.dom.classes.toggle(node, 'active');
TE._.dom.classes.get(node, 'active'); // check if `node` contains `active` class

Attribute

Add/remove and check HTML attributes:

TE._.dom.attributes.set(node, 'title', 'Test title.'); // set `title` attribute with value of `Test title.`
TE._.dom.attributes.set(node, {
    'title': 'Test title.',
    'rel': 'nofollow',
    'target': '_blank'
});

Setting an attribute with value of null will removes that attribute.

TE._.dom.attributes.reset(node, 'title'); // remove `title` attribute from `node`
TE._.dom.attributes.reset(node, ['title', 'rel', 'target']); // remove `title`, `rel` and `target` attributes from `node`
TE._.dom.attributes.get(node, 'title'); // get `title` attribute value from `node`
TE._.dom.attributes.get(node, ['title', 'rel', 'target']); // get `title`, `rel` and `target` attributes value from `node`

Data

Add/remove and check HTML data-* attributes:

TE._.dom.data.set(node, 'user', 'ta.tau.taufik'); // set `data-user` attribute with value of `ta.tau.taufik`
TE._.dom.data.set(node, {
    'user': 'ta.tau.taufik',
    'pass': 'foo bar baz'
});
TE._.dom.data.reset(node, 'user'); // remove `data-user` attribute from `node`
TE._.dom.data.reset(node, ['user', 'pass']); // remove `data-user` and `data-pass` attributes from `node`
TE._.dom.data.get(node, 'user'); // get `data-user` attribute value from `node`
TE._.dom.data.get(node, ['user', 'pass']); // get `data-user` and `data-pass` attributes value from `node`

Offset

Get node offset relative to its parent:

console.log(TE._.dom.offset(node)); // return `{"l": 40, "t": 20}`

Coordinate

Get current mouse pointer offset relative to node:

node.addEventListener("mousemove", function(e) {
    console.log(TE._.dom.pointer(this, e)); // return `{"x": 4, "y": 8}`
}, false);

Size

Get outer width and outer height of node:

console.log(TE._.dom.size(node)); // return `{"w": 100, "h": 50}`

Content

TE._.dom.content.set(node, 'foo');

TE._.dom.content.reset(node);

console.log(TE._.dom.content.get(node));

var parent = TE._.dom.parent(node);

Events

Set

Set a click event to node:

function do_click(e) {
    console.log(e.target);
}

TE._.events.set("click", node, do_click);

Each event set will also set a new internal hook data named as on:%s (on:click in this case).

Reset

Remove a click event with do_click as the callback function from node:

TE._.events.reset("click", node, do_click);

Fire

Programmatically trigger click events attached to node:

TE._.events.fire("click", node, [ … ]); // `[ … ]` is for function parameter data

Hooks

Set

Set a new focus.textarea hook:

editor._.hooks.set("focus.textarea", function() {
    console.log('focus');
});

Set a new focus.textarea hook with ID of foo:

editor._.hooks.set("focus.textarea", function() {
    console.log('focus');
}, 'foo');

Reset

Remove all focus.textarea hooks:

editor._.hooks.reset("focus.textarea");

Remove a focus.textarea hook with ID of foo:

editor._.hooks.reset("focus.textarea", 'foo');

Fire

Trigger all focus.textarea hooks:

editor._.hooks.fire("focus.textarea", [ … ]); // `[ … ]` is for function parameter data

Trigger a focus.textarea hook with ID of foo:

editor._.hooks.fire("focus.textarea", [ … ], 'foo');

These methods are not static because every GUI instances will have their own hooks storage.
